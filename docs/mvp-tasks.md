# MVP Task List

Tasks to reach MVP (PRD Milestones 1‚Äì5). Each task is scoped for a single agent session. Tasks are ordered by dependency ‚Äî later tasks depend on earlier ones.

The guiding principle: **one museum end-to-end before adding complexity.** The Met is the vertical slice that validates the entire pipeline. Brooklyn and Harvard stress-test normalization. Authority data is built from real ingested data, not guesses.

Phase-0-source handoffs live at `docs/handoff-<source>-*.md` (e.g. `docs/handoff-dodson-hilton-next-chunk.md`, `docs/handoff-phase-0-transcription.md`) and persist across sessions. Ad-hoc session handoffs may also be written at `docs/handoff-session-*.md` when a session ends with significant transient state; these are deleted by the next session that consumes them.

#### MVP "good enough" breakpoint ‚Äî Phase-0 done criteria

Phase-0 source acquisition is considered **MVP-complete** and unblocks the start of Phase A (authority curation ‚Üí enrich assets ‚Üí beta launch) when **all of the following are true**:

**Must-land before Phase A can start (MVP-blocking):**

0. **Schema-integrity gate.** A source does not count as `‚úÖ` complete for Phase-A purposes while its **schema** ‚Äî the typed fields in its `reconciled.jsonl` ‚Äî carries an unresolved **P1 schema-design defect**. Three specific classes qualify: (a) **field-level rule-4 violation** ‚Äî a single typed field encoding two unrelated concepts (e.g., the Leprohon `stage_suffix` overload covering both same-king titulary stages and distinct queen-consort sub-entries); (b) **schema-level rule-3 violation** ‚Äî a field invariant, type constraint, or cross-field contract that exists only in README / prompt / `transcribe.md` prose with no enforcing test (not "a test is missing somewhere", but specifically a test that would enforce a field-level semantic the schema claims to hold); (c) a **typed field shape that demonstrably cannot represent the data Leprohon/Baud/etc. prints faithfully** ‚Äî forces the extractor into a silent lossy encoding. A source hitting any of those three must be flagged `üü° provisionally complete pending schema PR` with the open schema work recorded on its row; "substantively complete" is NOT a synonym and must not be used to cover a known schema-design defect. The schema PR either ships first or the source holds the provisional flag until it does.

   **Out of scope for this gate:** content-level P1s do NOT demote `‚úÖ` under this gate. That includes parser / merge / `fix_rows` bugs that produce wrong values in correctly-shaped fields (pharaoh.se Roman date signs, Caracalla index-vs-page shift, Baud's missed eldest-son role-derivation rule), merge-logic defects that admit arbitrary rows (Ryholt's no-majority singleton admission ‚Äî **fixed across all 7 source-specific `merge.py` files in PR for issue #114, 2026-04-25; structural test in `tests/test_authority_merge_singleton_rejection.py`**), and Rule-1 provenance gaps (Kitchen's uncommitted source artifacts). These are real P1s ‚Äî several are named in the sweep audit ‚Äî but they are **tracked separately** under the **Known blind-spot audits** section below as pre-Phase-A blockers; they are fixable within a source without changing its schema, and they do not compound through Phase-A consumers the way schema debt does.

   **Rationale.** Schema debt on a primary authority compounds through Phase-A consumers in a way content bugs don't. A wrong Caracalla date in pharaoh.se is one fix in one source; a `stage_suffix` overload propagates into the shape of `rulers.json` and every downstream query that reads it. The cost curve of schema fixes flips from "hours" at chunk-author time to "weeks + coordination" once `rulers.json` / `sites.json` start consuming the field. The 2026-04-23 Leprohon `stage_suffix` overload is the anchoring case (see post-mortem in session log).

**pharaoh.se narrowed role (post-Beckerath, Phase-A `build_rulers.py` consumer guidance):** pharaoh.se is now a **gap-fill secondary** scoped to (a) **Roman emperors** (30 rows; the only on-disk source for the imperial period), (b) **Argead / Ptolemaic** as a thin secondary cross-validator on top of Leprohon Ch X chunk 14 (19 rows; redundant with Leprohon for kings, useful for `alt_labels`), and (c) the **"unplaced kings" / Dyn-13‚Äì14 long-tail** where no other source attests the king at all (Ryholt 1997 is comprehensive for SIP main lines and for the Abydos Dynasty ‚Äî pharaoh.se carries a few additional ephemeral names not in Ryholt). Phase A's `build_rulers.py` should:

- Take **titulary** from Leprohon 2013 first, falling back to pharaoh.se ONLY when Leprohon is silent on a given king.
- Take **chronology** from the plural-named-chronologies map (Beckerath 1997 lead / HKW 2006 fallback / Kitchen 1996 for Dyn 21‚Äì26 finer grain). Drop pharaoh.se's `chronologies` dict from the merge **for rulers covered by these primary sources (Dyn 0‚Äì31, ending 332 BCE)** ‚Äî pharaoh.se is second-hand to the primary sources it cites (Beckerath, Shaw, etc.), and the project now uses more direct scholarly authorities. **Argead, Ptolemaic, Roman-emperor, and unplaced-king rows retain their chronology from pharaoh.se** ‚Äî the primaries don't cover the post-Alexander periods or ephemeral kings until Bagnall & Rathbone 2004 (Roman) and an Argead/Ptolemaic chronology source land.
- Take **predecessor/successor chains** from `sequence_in_dynasty` ordering across Beckerath/Leprohon/Ryholt/Kitchen rather than pharaoh.se's chains. pharaoh.se chains remain useful as a cross-validation signal but not as the source of truth.
- Take **Roman emperors entirely from pharaoh.se** until a primary Roman source lands.
- Take **`alt_labels`** from pharaoh.se where Leprohon doesn't already carry the variant. pharaoh.se covers the Greek/Latin variant graph (Akoris/Akhoris, Necho/Nekau, Psamtik/Psammetichos) that Leprohon does not exhaustively maintain.

This narrows pharaoh.se from "secondary across the full ruler list" to "gap-fill on specific clusters". The committed 381-row `reconciled.jsonl` does NOT shrink ‚Äî Phase-A consumers simply read fewer of its fields. The `fetch.py` ingestion remains supported so the data can be regenerated. **End state target:** drop pharaoh.se entirely once Bagnall & Rathbone 2004 lands (Roman emperors) AND the unplaced-king long-tail is curated into Leprohon's existing Abydos-Dynasty / problematic-king sections or accepted as a documented MVP gap.
