In Porter & Moss topographical volumes, a personal name is a **tomb-owner headword (emit a row)** only if PM sets it in **large/small caps** flush at the entry head. A name in **normal upper-lower case** sitting under an object sub-block (`Statues.`, `Blocks, etc.`, `False-doors.`, `Finds`, `Reliefs`) is a **catalogued find-object — exclude it**, even when it has a name + titles and even when the surrounding banner says "…TOMBS".

**Why:** chunk-36 Abûsîr (PM III.1 p.345) put Ipi / Sitimp / Khuenkhnum / Seshemnūfer under a `False-doors.` sub-block of the "DYNASTY VI AND FIRST INTERMEDIATE PERIOD TOMBS" banner. 2/3 extraction agents emitted them as tombs; they are normal-case object entries carrying Schäfer find-numbers [17]-[20] that continue the `Blocks, etc.` sequence [10]-[16] — whose names (Sienptaḥ, Shededu-isi, Kairi) the same agents correctly excluded. The genuine headwords right below (mR I. INEMAKHET, etc.) are in caps. So: exclude the four false-doors.

**How to apply:** two reliable tells that a name is an object not a headword — (1) it is NOT in caps; (2) it carries an excavator object-number in a running find-sequence and/or names a museum inventory number. When a whole PM sub-section (e.g. a destroyed-superstructure period group) contains ONLY object sub-blocks and no caps headwords, it yields zero rows regardless of the banner. Cross-check: are parallel object names in the same section being excluded? If yes, exclude these too. Related: [[project-pm-memphis-abusir]].
